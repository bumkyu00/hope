# Exp 2 심층 분석: QA 기억 인코딩의 일반화 한계

## 실험 개요

- 모델: Qwen3.5-4B + LoRA (rank=16, lr=2e-5, ~250 steps)
- 사용자 3명 × 10 사실 = 30 QA 학습
- 조건 A: QA only (30개), 조건 B: QA + Context 예시 (54개)
- 평가: 30개 직접 recall + 30개 간접 generalization (수동 검증)

## 정량 결과 (수동 판정)

|  | A (QA only) | B (QA + Context) |
|---|---|---|
| Recall | 87% (26/30) | 93% (28/30) |
| Generalization | **27%** (8/30) | **60%** (18/30) |

자동 키워드 판정은 recall=100%, gen=60~67%로 나왔으나, 수동 검증 시 실질적으로 더 낮음.

## 현상 1: 사용자 간 사실 혼동 (Cross-user Contamination)

가장 심각한 문제. 한 사용자의 사실이 다른 사용자 질문에 침범.

### 관찰된 혼동 패턴

| 질문 대상 | 기대 사실 | 실제 응답 | 침범 출처 |
|---|---|---|---|
| 김지현 취미 | 등산 | "등산과 수영" | 박민수(수영) |
| 김지현 새 프로젝트 | Python | "Rust" | 박민수(R→Rust 변형) |
| 박민수 운동 | 수영 | "등산, 북한산" | 김지현(등산) |
| 박민수 여행 | 런던 | "파리" | 이서연(파리) |
| 이서연 스트레스 | 요가 | "수영" | 박민수(수영) |
| 이서연 여행 (A) | 파리 | "교토" | 김지현(교토) |
| 이서연 반려동물 장난감 | 토끼 | "고양이 전용" | 김지현(고양이) |
| 김지현 펫용품 (B) | 고양이 | "견을 키우고 있어요" | 박민수(강아지) |

### 분석

**혼동 방향이 일정하지 않음.** 특정 사용자가 특정 사용자를 침범하는 패턴이 아니라, 카테고리 단위로 섞임. "반려동물" 카테고리에서 3명의 사실이 뒤섞이고, "취미" 카테고리에서도 마찬가지.

**원인 추정:** LoRA가 `(사용자, 카테고리) → 사실` 매핑이 아니라 `카테고리 → 사실들의 혼합`으로 학습. 즉 "취미"라는 컨텍스트가 활성화되면 3명의 취미가 모두 후보로 올라오고, 사용자 이름으로 정확히 필터링하지 못함.

**LoRA의 구조적 한계일 수 있음.** rank=16의 low-rank 공간에서 3명 × 10사실 = 30개의 독립적 바인딩을 유지하기엔 용량이 부족. 특히 같은 카테고리의 사실들이 비슷한 subspace를 공유하면서 간섭 발생.

## 현상 2: 알레르기/공포증 무시 (Safety-critical Failure)

가장 위험한 문제. 알레르기와 공포증은 실수하면 실제 피해가 발생하는 사실인데, 모델이 이를 무시하거나 정반대로 행동.

### A 조건에서 발생한 위반

| 사실 | 질문 | 응답 | 위험도 |
|---|---|---|---|
| 김지현 견과류 알레르기 | 간식 추천 | "견과류 믹스 + 견과유 쿠키" | 🔴 **치명적** |
| 김지현 채식주의자 | 생일파티 | "해산물 스테이크" | 🟠 |
| 김지현 발표 공포증 | 발표 제안 | "좋은 아이디어" | 🟠 |
| 박민수 유제품 알레르기 | 디저트 추천 | "채식 요거트" (유제품) | 🔴 **치명적** |
| 이서연 해산물 알레르기 | 초밥 추천 | "건강하고 깔끔" 긍정 추천 | 🔴 |
| 이서연 높은 곳 공포증 | 번지점프 | "흥미로운 경험" | 🟠 |

### B 조건에서의 개선

B 조건에서는 알레르기 위반이 **0건**으로 감소. Context 예시에 "견과류가 들어있는지 확인해야 합니다", "유제품은 피해야 합니다" 같은 적용 예시가 포함되었기 때문.

### 분석

QA만으로는 "김지현의 알레르기는? → 견과류"를 외워도, "간식 추천해줘"에서 이 사실을 **적용해야 한다는 연결고리**를 학습하지 못함. Context 예시가 이 연결고리("빵 선물해도 될까? → 견과류 확인해야 합니다")를 명시적으로 보여줘야 학습됨.

**시사점:** 안전 관련 사실은 반드시 적용 예시와 함께 학습시켜야 함. QA 외우기만으로는 부족.

## 현상 3: Hallucination 증가

학습하지 않은 사실을 자신있게 생성하는 현상.

### 관찰된 hallucination

| 응답 | 실제 | 유형 |
|---|---|---|
| "김지현은 Python, JavaScript, Rust" | Python만 | 사실 확장 |
| "이서연은 Figma, Notion, Linear" | Figma만 | 도구 확장 |
| "이서연은 요가 수련과 필라테스" | 요가만 | 취미 확장 |
| "김지현은 재즈 피아니스트로 활동" | 재즈를 좋아함 | 역할 hallucination |
| "교토대 컴퓨터과학 교수" | 소프트웨어 엔지니어 | 직업 hallucination |
| "박민수 비행기 발수건(Hand sanitizer)" | 비행기 공포증 | 의미 불명 |

### 분석

**A 조건에서 hallucination이 더 심함.** 짧은 QA 답변("Python")을 학습했지만, 생성 시 모델이 답변을 "풍부하게" 만들려다 학습하지 않은 사실을 추가.

B 조건에서는 Context 예시가 적절한 길이의 응답 패턴을 보여줘서 hallucination이 상대적으로 적음.

## 현상 4: 다국어 오염 (Multilingual Leakage)

### B 조건에서 관찰된 다국어 삽입

| 응답 내 외국어 | 맥락 |
|---|---|
| "김地現" | 한자 변환 |
| "週末可以一起爬山，比如北漢山。他經常去那裡。" | 중국어 문장이 통째로 |
| "유제품过敏" | 중국어 단어 |
| "每周 2~3 회 방문" | 중국어 단어 |
| "推荐给요" | 한중 혼합 |
| "了解ました" | 일본어 혼합 |
| "布艺나硅胶 소재" | 중국어 단어 |
| "높은 곳恐惧" | 중국어 단어 |

### 분석

**A 조건에서는 거의 없고 B 조건에서 심함.** Context 예시가 한국어로만 되어있는데 왜 B에서 더 심한가?

추정: B 조건의 학습 데이터가 더 많고(54 vs 30) 다양한 패턴을 포함하면서, 모델의 다국어 경계가 불안정해짐. 특히 "알레르기", "공포증" 같은 전문 용어 근처에서 중국어 토큰이 활성화되는 경향. Qwen 모델이 중국어 중심으로 사전학습되었기 때문에, LoRA 학습이 한국어 토큰 생성의 안정성을 일부 훼손하는 것으로 보임.

**심각도:** 의미 전달에는 대부분 지장 없지만, 프로덕션에서는 후처리 필요. 학습 데이터에 한국어-only 제약을 강화하거나, 생성 시 한국어 토큰에 bias를 줄 수 있음.

## 현상 5: 카테고리별 일반화 난이도 차이

### 일반화 성공률 by 카테고리 (B 조건)

| 카테고리 | 성공/전체 | 성공률 | 난이도 |
|---|---|---|---|
| 직업/도구 | 6/6 | 100% | 쉬움 |
| 알레르기 | 3/3 (B) | 100% | Context 있으면 쉬움 |
| 공포증 | 2/3 | 67% | 중간 |
| 여행지 | 2/3 | 67% | 중간 |
| 식습관 | 2/3 | 67% | 중간 |
| 음악 | 2/3 | 67% | 중간 |
| 취미/운동 | 1/3 | 33% | 어려움 |
| 반려동물 | 0/3 | 0% | 매우 어려움 |
| 선호 언어 | 0/1 | 0% | 매우 어려움 |

### 분석

**직업/도구가 가장 잘 됨.** "소프트웨어 엔지니어 → 코드 관련", "Figma → 디자인 관련"은 사전학습에서 이미 강한 연결이 존재. LoRA가 이 연결을 사용자별로 특화만 하면 됨.

**반려동물이 가장 안 됨.** "고양이 → 펫용품" 연결은 사전학습에 있지만, 3명의 반려동물(고양이/강아지/토끼)을 사용자별로 구분하는 것이 어려움. rank=16 LoRA 공간에서 "반려동물 종류"에 대한 사용자별 분리가 안 되는 것.

**가설:** 사전학습에서 이미 강한 연결이 있는 카테고리(직업→업무, 알레르기→음식 주의)는 LoRA가 사용자 특화만 하면 되므로 쉬움. 사전학습에서 약한 연결인 카테고리(반려동물→선물, 선호 언어→보고서 언어)는 LoRA가 연결 자체를 새로 만들어야 하므로 어려움.

## 현상 6: A vs B 질적 차이

### A의 특징: "모르면 일반론으로 답변"

A 조건은 사실을 recall하지 못하면 사전학습 지식에 의존한 일반적 답변을 생성:
- "주말에 뭐 할까?" → 교토 카페 추천 (등산이 아니라 "교토"만 기억)
- "운동 추천" → "재즈 발라드 들으며 요가" (사실과 무관한 일반론)
- "보고서 언어" → "한국어" (상식적 답변)

### B의 특징: "외운 패턴을 재활용"

B 조건은 Context 예시에서 학습한 응답 패턴을 적극 재사용:
- "Python으로 쓸 거예요. 김지현은 Python을 주로 사용합니다." (Context 예시 거의 복사)
- "비행기 공포증이 있어서 부담스러울 수 있어요. 원격 작업이 가능한지 확인하세요." (Context 예시 패턴)

**장점:** 사실이 정확히 적용됨
**단점:** Context 예시에 없는 새로운 적용 방식은 만들지 못함. 틀에 박힌 응답.

## 결론 및 시사점

### 1. QA 외우기는 필요조건이지 충분조건이 아니다
사실을 100% recall해도 새로운 맥락에 적용하는 건 별개의 능력. Context 예시가 반드시 필요.

### 2. Context 예시는 "연결고리"를 학습시킨다
"채식주의자" → "점심 추천 시 채식 메뉴"라는 연결고리는 명시적으로 보여줘야 학습됨. 이것이 dream replay의 핵심 역할.

### 3. 사용자 구분이 핵심 병목이다
현재 가장 큰 실패 원인은 사실 자체를 모르는 것이 아니라, 사실을 올바른 사용자에게 귀속시키지 못하는 것. rank 증가, 사용자별 LoRA 분리, 또는 system prompt에 사용자 ID 명시 등의 접근 필요.

### 4. 안전 관련 사실은 특별 처리가 필요하다
알레르기, 공포증은 실패 시 실질적 피해가 발생. 이런 사실은 다양한 적용 예시를 더 많이 학습시키거나, 생성 시 명시적으로 체크하는 메커니즘 필요.

### 5. 다국어 모델의 fine-tuning 안정성 문제
Qwen의 중국어 기반 사전학습이 한국어 fine-tuning 시 불안정성을 유발. 특히 B 조건(더 많은 학습)에서 심화. 한국어-only 생성 제약 또는 다국어 penalty 필요.

### 6. CMS의 기대 역할
현재 실험은 모든 사실을 한 번에 학습. CMS는 시간에 걸쳐 점진적으로 학습하면서 저주파 층에 사용자-사실 바인딩을 공고화할 수 있음. 특히 반복되는 패턴(매번 Python 사용)이 저주파 층에 안착하면 사용자 혼동이 줄어들 수 있다는 가설. Phase 1.5에서 검증 필요.
